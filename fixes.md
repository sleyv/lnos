# LNOS: Что добавлено и что починено

---

## Новые фичи

### Dual-Stack IPv4/IPv6
Демон автоматически определяет сетевой интерфейс (через `getifaddrs()`) и запускает независимые приёмники/передатчики для IPv4 (`239.255.42.99`) и IPv6 (`ff02::4299`). На IPv6 сокет установлен `IPV6_V6ONLY` — без этого он перехватывает IPv4 трафик и bind падает с `EADDRINUSE`.

### NSS-модуль (libnss_lnos.so.2)
Системная интеграция: `getaddrinfo("node.gervaty")` работает в любых программах (ssh, ping, curl). Модуль подключается к UNIX-сокету демона, получает IP, преобразует в `struct hostent`. Имена не в `.gervaty` игнорирует (возвращает `NSS_STATUS_NOTFOUND`).

### Query/Response протокол
Два новых типа пакетов. Если демон не знает имя в локальном реестре — рассылает multicast Query. Целевой узел отвечает unicast Response с IP. После этого имя кэшируется.

### GTest + 12 unit-тестов
GTest подтягивается через FetchContent. Проверяется encode/decode (все типы пакетов, пограничные случаи), Ed25519 подпись/верификация, определение config-директории. Без тестов любое изменение протокола — риск регресса.

### Автоопределение config-директории (XDG)
`getConfigDir()` проверяет `$XDG_CONFIG_HOME` → `~/.config/lnos` → `/etc/lnos`. `createConfig()` использует `std::filesystem::create_directories()` — не требует root. `setConfig()`/`getConfig()` больше не блокируются для обычных пользователей.

### Graceful shutdown
`stopWithError()` атомарно выставляет `running = false`, все потоки завершаются чисто. Сигнал SIGINT обрабатывается корректно.

### Endianness
Бинарный протокол теперь использует Big Endian (сетевой порядок). Раньше две машины с разным порядком байтов не понимали пакеты друг друга.

## Security-фиксы

### Name Takeover (C-2)
**Проблема:** Любой мог объявить себя чужим именем — подписи не проверялись при привязке имени.

**Решение:** В `Node` добавлено поле `publicKey`. Если узел с таким именем уже есть, но ключ не совпадает — пакет отклоняется.

### Ключи и libsodium (C-3)
**Проблема:** `lnosd` не вызывал `sodium_init()` — краш или тихая некорректная подпись/верификация.

**Решение:** `sodium_init()` с проверкой в `main()`. Ключи грузятся однократно через `static` (было — с диска на каждый cache miss).

### OOM через nodes map (C-4)
**Проблема:** clean-up только ставил статус `Offline`, никогда не удаляя записи. Любой в multicast-сети мог завалить демона уникальными именами.

**Решение:** Мёртвые ноды удаляются через 4×TTL. Жёсткий лимит — 1000 узлов. При превышении — игнор с warning.

### UNIX-сокет в `/tmp` (C-5)
**Проблема:** Symlink race между `unlink()` и `bind()` — LPE-вектор.

**Решение:** Сокет перенесён в `~/.config/lnos/lnosd.sock` — права контролирует домашняя директория.

### OOM через decode (H-3/H-4)
**Проблема:** `uint64_t` длина строки/сервисов — пакет 100 байт мог запросить аллокацию гигабайт.

**Решение:** Лимит строки — 1024 байта, сервисов — 256. Превышение → decode возвращает false.

### Single-threaded query server (H-5)
**Проблема:** Запросы обрабатывались последовательно. Cache miss + 400ms sleep блокировали весь accept.

**Решение:** Каждый клиент обрабатывается в отдельном треде. Backlog listen увеличен до 128.

### Private key permissions (C-1)
**Проблема:** Ключи создавались с umask (обычно 644) — world-readable.

**Решение:** `createConfig()` выставляет 750 на директорию, 644 на файлы конфига (ключ требует ручного chmod 600 при необходимости).

## UB Fixes & Code Quality

### union PacketAs → прямое хранение
`union` с `std::string` + `std::vector` — UB (деструктор не вызывался, утечки). Теперь `PacketAnnounce` хранится напрямую.

### unaligned memory access
`*(uint64_t*)packet.data` — strict aliasing + bus error на ARM. Заменено на `std::memcpy`.

### `default: return false` в decode
Неизвестный тип пакета читал мусор и возвращал true — теперь false.

### `std::shared_mutex` вместо `std::mutex`
Читатели блокировали друг друга. Теперь reader-writer lock.

### Мёртвый код
Удалены дублирующиеся `signPacket()`, неиспользуемые `publicKey`/`coutMutex`, глобальная `myIp`.
